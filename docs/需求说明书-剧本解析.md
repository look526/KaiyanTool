# 需求说明书 - 剧本解析

## 一、功能概述

剧本解析功能通过 AI 智能分析剧本内容，自动提取结构化数据，包括场景、角色、物品、分镜等信息，为后续资产生成和视频制作提供数据基础。

---

## 二、解析输出结构

### 2.1 场景 (Scene)

每个场景包含以下字段：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 场景唯一标识 | `scene_1` |
| `number` | number | 场景序号 | `1` |
| `heading` | string | 场景标题 | `咖啡馆内 - 下午` |
| `location` | string | 地点名称 | `咖啡馆` |
| `time` | string | 时间设定 | `下午`、`夜晚`、`黄昏` |
| `description` | string | 场景描述（150-300字） | 包含环境氛围、空间布局、光影效果、细节装饰 |
| `characters` | string[] | 出场角色列表 | `["张三", "李四"]` |
| `dialogues` | Dialogue[] | 对话列表 | 详见 2.2 |
| `actions` | Action[] | 动作列表 | 详见 2.3 |
| `items` | Item[] | 物品列表 | 详见 2.5 |

---

### 2.2 对话 (Dialogue)

每段对话包含：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `characterName` | string | 说话角色名称 | `张三` |
| `text` | string | 对话内容 | `你好，好久不见！` |
| `shot` | Shot | 镜头信息 | 详见 2.4 |

---

### 2.3 动作 (Action)

每个动作包含：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `description` | string | 动作描述 | `张三起身走向窗边` |
| `type` | string | 动作类型 | `action` |
| `shot` | Shot | 镜头信息 | 详见 2.4 |

---

### 2.4 镜头 (Shot)

每个镜头包含以下信息：

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `type` | string | 镜头类型 | `特写`、`近景`、`中景`、`全景`、`远景` |
| `movement` | string | 镜头运动 | `推`、`拉`、`摇`、`移`、`跟`、`固定` |
| `angle` | string | 镜头角度 | `平视`、`俯视`、`仰视`、`斜角` |
| `description` | string | 镜头描述 | 描述如何拍摄、角色动作、表情、环境互动 |
| `duration` | number | 预估时长（秒） | `3-10` |
| `transition` | string | 转场方式 | `切`、`淡入淡出`、`溶解`、`划变` |

#### 镜头类型选择指南

| 类型 | 适用场景 | 拍摄范围 |
|------|----------|----------|
| 特写 | 表情细节、物品细节、情感强调 | 面部或物品局部 |
| 近景 | 对话、表情展示 | 胸部以上 |
| 中景 | 日常对话、动作展示 | 膝盖以上 |
| 全景 | 角色全身、环境互动 | 全身及周围环境 |
| 远景 | 环境展示、场景氛围 | 整体环境为主 |

#### 镜头运动选择指南

| 运动 | 适用场景 | 效果 |
|------|----------|------|
| 推 | 强调情感、突出重点 | 聚焦注意力 |
| 拉 | 展示环境、结束场景 | 扩展视野 |
| 摇 | 跟随视线、展示空间 | 空间转换 |
| 移 | 平行跟随、展示过程 | 动态展示 |
| 跟 | 跟随角色移动 | 保持连贯 |
| 固定 | 静态场景、对话 | 稳定画面 |

---

### 2.5 物品 (Item)

每个物品包含：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `name` | string | 物品名称 | `咖啡杯` |
| `size` | string | 尺寸规格 | `10cm×8cm×5cm` |
| `shape` | string | 形状特征 | `圆柱形` |
| `color` | string | 颜色属性 | `白色，带蓝色花纹` |

---

### 2.6 角色 (Character)

每个角色包含：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 角色唯一标识 | `char_1` |
| `name` | string | 角色名称 | `张三` |
| `description` | string | 角色描述（简短介绍） | `男主角，程序员` |
| `appearance` | object | 外貌信息 | 见下方 |
| `costume` | object | 服装信息 | 见下方 |
| `personality` | string[] | 性格特质 | `["开朗（总是面带微笑）", "细心（注意细节）"]` |
| `lines` | number | 对话行数 | `15` |

#### 外貌信息 (appearance)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `hairStyle` | string | 发型样式 | `短发，黑色` |
| `facialFeatures` | string | 五官特征 | `圆脸，大眼睛，高鼻梁` |
| `bodyProportion` | string | 身材比例 | `身材高挑，体型偏瘦` |
| `otherDetails` | string[] | 其他外貌细节 | `["戴黑框眼镜", "左手戴手表"]` |

#### 服装信息 (costume)

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `type` | string | 服装类型 | `休闲西装` |
| `color` | string | 主体颜色 | `深蓝色` |
| `material` | string | 材质特性 | `棉质` |
| `decoration` | string | 特殊装饰 | `银色纽扣` |

#### 外貌描写生成规则

生成资产时，将以下属性组合成完整的外貌描述：

```
{hairStyle}，{facialFeatures}，{bodyProportion}，{otherDetails.join('，')}，穿着{type}、{color}、{material}、{decoration}
```

**示例输出**：
```
短发，黑色，圆脸，大眼睛，高鼻梁，身材高挑，体型偏瘦，戴黑框眼镜，左手戴手表，穿着休闲西装、深蓝色、棉质、银色纽扣
```

**注意**：
- 外貌描写只包含视觉特征
- 不包含年龄、职业、性格等非视觉信息
- 服装信息以"穿着"开头

## 四、数据流程

```
用户输入剧本
    ↓
点击"解析剧本"按钮
    ↓
选择解析模式（快速解析 / AI智能解析）
    ↓
显示处理进度
    ↓
调用 AI 解析服务
    ↓
返回结构化数据
    ↓
切换到预览模式展示结果
    ↓
用户确认后点击"生成资产"
    ↓
创建角色、物品、场景、分镜到数据库
```

---

## 五、资产生成规则

### 5.1 角色资产

- **名称**：角色名称
- **外貌描写**：纯视觉特征描述（发型、五官、身材、服装）
- **年龄**：默认 25 岁（如未指定）
- **性别**：根据名字和描述推断

### 5.2 物品资产

- **名称**：物品名称
- **描述**：尺寸 + 形状 + 颜色组合

### 5.3 场景资产

- **地点**：场景地点
- **时间**：场景时间
- **氛围**：场景描述

### 5.4 分镜资产

- **场景关联**：所属场景 ID
- **序号**：分镜顺序
- **描述**：镜头描述 + 对话/动作内容
- **时长**：预估时长
- **镜头运动**：镜头类型 + 运动 + 角度
- **起始提示词**：完整的画面描述（用于 AI 生成图像）
- **结束提示词**：结束画面描述

---

## 六、注意事项

1. **外貌描写必须纯视觉**：只描述可见的外表特征，不包含年龄、职业、性格等抽象信息
2. **场景描述长度**：控制在 150-300 字，包含环境氛围、空间布局、光影效果
3. **镜头信息完整性**：每个对话和动作都必须有对应的镜头信息
4. **物品信息完整**：必须包含名称、尺寸、形状、颜色四个属性
5. **角色去重**：同名角色自动合并，累加对话行数
6. **物品去重**：同名物品自动合并，记录出现场景
