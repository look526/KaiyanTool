# 登录注册功能需求文档

## 1. 功能需求

### 1.1 注册功能
- **用户注册**：支持用户通过邮箱、用户名和密码进行注册
- **输入验证**：
  - 用户名：2-50个字符
  - 邮箱：符合邮箱格式
  - 密码：至少8位，包含至少1个大写字母、1个小写字母、1个数字、1个特殊字符
- **唯一性检查**：
  - 邮箱必须唯一
  - 用户名必须唯一
- **密码加密**：使用bcrypt算法对密码进行加密存储
- **注册成功后**：自动登录并跳转到项目列表页

### 1.2 登录功能
- **用户登录**：支持通过邮箱和密码登录
- **输入验证**：
  - 邮箱：符合邮箱格式
  - 密码：至少6位
- **记住我**：支持30天免登录功能
- **登录尝试限制**：
  - 连续登录失败会临时锁定账户
  - 锁定时间：15分钟
- **登录成功后**：跳转到项目列表页

### 1.3 登出功能
- **用户登出**：清除会话并跳转到登录页
- **会话清理**：删除数据库中的会话记录
- **Cookie清理**：清除sessionId cookie

### 1.4 用户信息获取
- **获取当前用户**：通过会话获取当前登录用户信息
- **会话验证**：检查会话是否过期
- **会话更新**：自动延长会话有效期

## 2. 非功能需求

### 2.1 安全性
- **密码加密**：使用bcryptjs进行密码哈希
- **会话管理**：使用安全的sessionId cookie
- **登录限制**：防止暴力破解攻击
- **HTTPS**：生产环境使用HTTPS
- **CSRF保护**：防止跨站请求伪造

### 2.2 性能
- **响应时间**：登录/注册请求响应时间不超过2秒
- **会话管理**：高效的会话查询和更新

### 2.3 可用性
- **错误处理**：提供清晰的错误提示
- **用户体验**：流畅的登录注册流程
- **会话过期**：会话过期后自动跳转到登录页

## 3. 数据需求

### 3.1 数据库表
- **User表**：
  - id (主键)
  - name (用户名)
  - email (邮箱)
  - password_hash (密码哈希)
  - avatar_url (头像URL)
  - role (角色)
  - created_at (创建时间)

- **Session表**：
  - id (主键)
  - token (会话令牌)
  - user_id (用户ID)
  - expires_at (过期时间)

### 3.2 数据传输
- **注册请求**：
  ```json
  {
    "name": "用户名",
    "email": "邮箱",
    "password": "密码"
  }
  ```

- **登录请求**：
  ```json
  {
    "email": "邮箱",
    "password": "密码",
    "rememberMe": true/false
  }
  ```

- **响应格式**：
  ```json
  {
    "user": {
      "id": "用户ID",
      "name": "用户名",
      "email": "邮箱",
      "avatar_url": "头像URL",
      "role": "角色",
      "created_at": "创建时间"
    },
    "rememberMe": true/false
  }
  ```

## 4. 技术实现

## 5. API接口

### 5.1 注册接口
- **路径**：`POST /auth/register`
- **请求体**：注册信息
- **响应**：201 Created + 用户信息

### 5.2 登录接口
- **路径**：`POST /auth/login`
- **请求体**：登录信息
- **响应**：200 OK + 用户信息

### 5.3 登出接口
- **路径**：`POST /auth/logout`
- **响应**：200 OK + 退出成功消息

### 5.4 获取当前用户接口
- **路径**：`GET /auth/me`
- **响应**：200 OK + 用户信息

### 5.5 会话更新接口
- **路径**：`POST /auth/update-session`
- **响应**：200 OK + 会话更新成功消息
