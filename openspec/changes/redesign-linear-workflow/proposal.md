# Proposal: Redesign Linear Workflow

## Summary

重新设计 kaiyanTool 的核心工作流，构建一个以线性导航为主导的内容创作系统。通过全新的 UI 设计和功能整合，实现从剧本创作到分镜制作的完整线性工作流，确保角色、场景、分镜之间的高度一致性。

## Problem Statement

### 当前问题

1. **工作流割裂**：现有功能模块（剧本编辑、角色管理、场景管理、分镜管理）相对独立，用户需要在多个页面间跳转，缺乏清晰的操作路径指引。

2. **角色一致性缺失**：角色描述与生成的图片之间缺乏强制关联，导致分镜中的角色形象可能不一致。

3. **数据流转不畅**：剧本解析后，场景、角色信息未能有效传递到后续的分镜制作环节。

4. **入口分散**：小说输入和剧本输入作为两个独立入口，增加了用户认知负担。

5. **UI 设计陈旧**：现有页面布局未能体现创作流程的逻辑性，用户体验不够直观。

### 用户痛点

- "我不知道下一步该做什么"
- "生成的角色图片和描述对不上"
- "改了剧本，分镜里的信息没有同步更新"
- "小说和剧本为什么要分开两个地方？"

## Goals

### 主要目标

1. **建立线性工作流**：设计以 Sidebar 为核心的线性导航系统，用户从上到下自然完成剧本→角色→场景→分镜的创作流程。

2. **确保角色一致性**：采用"定妆照优先"策略，用户必须先生成角色定妆照，后续所有生成都基于此参考图。

3. **统一内容入口**：将小说作为剧本的一种输入格式，统一到剧本编辑器中。

4. **全新 UI 设计**：重新设计整个界面，突出工作流的逻辑性和连贯性。

### 非目标

- 不改变现有的后端 API 结构（保持兼容）
- 不涉及用户认证和权限系统的改动
- 不改变 AI 模型集成方式

## Scope

### In Scope

| 模块 | 描述 |
|------|------|
| 工作流导航 | 全新的 Sidebar 设计，线性步骤引导 |
| 剧本管理 | 统一的剧本编辑器，支持小说导入转换 |
| 场景解析 | 智能场景解析，自动提取角色和场景信息 |
| 角色一致性 | 定妆照生成系统，角色引用机制 |
| 分镜系统 | 基于场景和角色的分镜制作 |
| UI 重设计 | 全新的页面布局和交互设计 |

### Out of Scope

- 用户认证系统改动
- 后端 API 重构
- AI 模型集成变更
- 导出功能改动
- 视频生成逻辑改动

## Key Decisions

### Decision 1: 线性导航模式

**决策**：采用 Sidebar 线性导航，用户可自由修改每个步骤。

**理由**：
- 线性流程符合内容创作的自然逻辑
- 用户可随时返回修改，不强制顺序
- Sidebar 提供清晰的进度指示

**替代方案**：
- 强制步骤引导：过于限制用户自由度
- 完全自由导航：缺乏流程指引

### Decision 2: 定妆照优先策略

**决策**：角色必须先生成定妆照，后续生成基于定妆照参考。

**理由**：
- 确保角色视觉一致性
- 减少用户反复调整的工作量
- 提供明确的角色定义标准

**实现方式**：
- 角色创建时引导生成定妆照
- 分镜生成时自动引用角色定妆照
- 提供定妆照版本管理

### Decision 3: 小说转剧本统一入口

**决策**：将小说作为剧本的一种输入格式，统一到剧本编辑器。

**理由**：
- 减少用户认知负担
- 小说和剧本本质都是内容输入
- 简化导航结构

**实现方式**：
- 剧本编辑器支持"导入小说"功能
- AI 自动将小说转换为剧本格式
- 用户可编辑转换结果

### Decision 4: 全新 UI 设计

**决策**：重新设计整个界面，而非渐进式改进。

**理由**：
- 现有设计无法支撑新的工作流理念
- 全新设计可确保一致性和连贯性
- 避免新旧设计冲突

## Impact Analysis

### 用户影响

| 用户类型 | 影响 |
|---------|------|
| 新用户 | 更容易上手，清晰的流程指引 |
| 现有用户 | 需要适应新界面，但工作流更高效 |
| 高级用户 | 可快速跳转，不受限制 |

### 技术影响

| 系统 | 影响 |
|------|------|
| 前端 | 大规模重构，新建页面和组件 |
| 后端 | 轻微调整，主要是 API 响应格式优化 |
| 数据库 | 新增角色定妆照关联表 |
| AI 服务 | 新增小说转剧本 API |

### 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 用户适应成本 | 中 | 提供新用户引导和迁移教程 |
| 开发周期长 | 中 | 分阶段交付，优先核心功能 |
| 数据迁移问题 | 低 | 保持数据结构兼容 |
