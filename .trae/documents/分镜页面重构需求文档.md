# 分镜页面重构需求文档

## 1. 项目概述

### 1.1 背景
当前系统的分镜页面需要重构，以提供更直观、更高效的用户体验。新的分镜页面将采用剧集-场景-分镜的三层结构，支持AI辅助生成、@提及功能、视频生成备选等核心功能。

### 1.2 目标
- 提供直观的剧集-场景-分镜层级管理
- 支持AI辅助生成分镜
- 实现高效的@提及功能
- 提供视频生成和备选机制
- 支持草稿保存和历史记录

---

## 2. 用户故事

### 2.1 剧集管理
**作为**内容创作者  
**我想要**在一个页面查看所有剧集的概览  
**以便**快速了解项目进度和选择要编辑的剧集

**验收标准：**
- 显示剧集卡片网格
- 每个卡片显示剧集名称、场景数量、分镜数量
- 点击卡片进入分镜详情页面
- 支持创建新剧集
- 支持删除剧集

### 2.2 分镜详情查看
**作为**内容创作者  
**我想要**在分镜详情页面同时查看剧本和分镜  
**以便**在创作分镜时参考剧本内容

**验收标准：**
- 左侧显示完整剧本
- 顶部有场景下拉选择器
- 选择场景后高亮对应剧本内容
- 右侧显示分镜条目列表

### 2.3 分镜条目编辑
**作为**内容创作者  
**我想要**为每个分镜配置模型、长宽、分辨率等参数  
**以便**生成符合要求的视频

**验收标准：**
- 每条分镜显示模型选择器
- 支持选择长宽比例
- 支持选择分辨率
- 支持输入剧本描述
- 显示涉及的角色和物品

### 2.4 @提及功能
**作为**内容创作者  
**我想要**在描述框中使用@引用角色、物品、场景或素材库图片  
**以便**快速添加相关资源到描述中

**验收标准：**
- 输入@符号后弹出建议列表
- 建议列表包含角色、物品、场景、素材库图片
- 选择后以标签形式显示
- 支持删除已添加的标签

### 2.5 视频生成
**作为**内容创作者  
**我想要**为每个分镜生成多个备选视频  
**以便**选择最满意的结果

**验收标准：**
- 点击生成按钮调用AI模型
- 生成的视频存入素材库
- 显示在备选区
- 支持生成多个备选
- 可以选择一个为推荐

### 2.6 草稿和历史
**作为**内容创作者  
**我想要**保存未完成的配置并查看生成历史  
**以便**随时恢复工作状态

**验收标准：**
- 支持保存未完成配置
- 支持恢复草稿
- 保留生成历史记录
- 显示每个分镜的生成次数

## 3. 功能需求

### 3.1 剧集管理

#### 3.1.1 剧集列表
- 显示所有剧集卡片
- 每个卡片显示：
  - 剧集名称
  - 场景数量
  - 分镜数量
  - 创建时间
- 支持搜索剧集
- 支持按创建时间排序

#### 3.1.2 剧集创建
- 输入剧集名称
- 输入剧集描述（可选）
- 自动分配剧集编号
- 关联剧本（可选）

#### 3.1.3 剧集编辑
- 修改剧集名称
- 修改剧集描述
- 关联/修改剧本

#### 3.1.4 剧集删除
- 删除剧集前确认
- 级联删除关联的场景和分镜

### 3.2 分镜详情页面

#### 3.2.1 剧本查看器
- 显示完整剧本内容
- 支持滚动浏览
- 场景选择器切换场景
- 选择场景后高亮对应剧本段落
- 显示场景标记（地点、时间）

#### 3.2.2 分镜条目列表
- 按场景分组显示分镜
- 支持拖拽排序
- 支持批量操作
- 显示分镜状态（待生成、生成中、已完成）

#### 3.2.3 分镜条目编辑
- 模型选择器（从AI提供商配置中选择）
- 长宽选择器（16:9、9:16、4:3、1:1）
- 分辨率选择器（720p、1080p、2K、4K）
- 描述输入框（支持@提及）
- 显示关联的角色标签
- 显示关联的物品标签

### 3.3 @提及功能

#### 3.3.1 提及资源类型
- 角色（Character）
- 物品（Item）
- 场景（Scene）
- 素材库图片（Asset）

#### 3.3.2 提及交互
- 输入@符号触发建议列表
- 实时搜索匹配资源
- 显示资源名称和图标
- 选择后插入标签
- 标签可删除

#### 3.3.3 提及范围
- 只能引用当前项目的资源
- 素材库图片按项目隔离

### 3.4 视频生成

#### 3.4.1 单个生成
- 选择模型参数
- 点击生成按钮
- 显示生成进度
- 生成完成后显示视频预览

#### 3.4.2 批量生成
- 选择多个分镜
- 统一设置参数
- 批量生成视频
- 显示整体进度

#### 3.4.3 备选视频
- 每个分镜最多保留10个备选视频
- 显示备选视频列表
- 支持预览备选视频
- 支持删除备选视频
- 支持设置推荐视频

#### 3.4.4 推荐机制
- 推荐视频仅作为标记
- 不影响后续生成
- 支持切换推荐

### 3.5 草稿和历史

#### 3.5.1 草稿保存
- 自动保存（每30秒）
- 手动保存
- 保存所有未完成的配置

#### 3.5.2 草稿恢复
- 显示草稿列表
- 恢复指定草稿
- 删除草稿

#### 3.5.3 生成历史
- 记录每次生成的参数
- 记录生成结果
- 显示生成时间
- 支持查看历史详情

## 4. 非功能需求

### 4.1 性能要求
- 页面加载时间 < 2秒
- @提及响应时间 < 500ms
- 视频生成不阻塞UI
- 支持大量分镜（100+）流畅操作

### 4.2 可用性要求
- 遵循Glassmorphism设计规范
- 深色/浅色模式切换正常
- 响应式布局（支持桌面、平板）
- 操作反馈及时

### 4.3 兼容性要求
- 支持Chrome、Firefox、Safari最新版本
- 支持Edge最新版本
- 支持移动端浏览器

### 4.4 安全性要求
- 所有API需要认证
- 资源访问权限控制
- 防止XSS攻击
- 防止CSRF攻击

### 4.5 可维护性要求
- 代码遵循项目规范
- 组件化设计
- 类型定义完整
- 代码注释清晰

## 5. 数据需求

### 5.1 数据模型

#### 5.1.1 Episode（剧集）
- id: UUID
- project_id: UUID
- title: String
- episode_number: Int
- description: String?
- script_id: UUID?
- created_at: DateTime
- updated_at: DateTime

#### 5.1.2 ShotAlternative（分镜备选）
- id: UUID
- shot_id: UUID
- video_url: String
- is_recommended: Boolean
- version: Int
- metadata: Json?
- created_at: DateTime

#### 5.1.3 ShotDraft（分镜草稿）
- id: UUID
- shot_id: UUID?
- episode_id: UUID
- scene_id: UUID?
- draft_data: Json
- created_at: DateTime
- updated_at: DateTime

### 5.2 数据关系
- 1个Project → 多个Episode
- 1个Episode → 多个Scene
- 1个Scene → 多个Shot
- 1个Shot → 多个ShotAlternative
- 1个Episode → 多个ShotDraft

### 5.3 数据隔离
- 素材库按项目隔离
- @提及只能引用当前项目资源
- 用户只能访问有权限的项目

## 7. API需求

### 7.1 Episode API
- GET /projects/:projectId/episodes - 获取剧集列表
- POST /projects/:projectId/episodes - 创建剧集
- GET /episodes/:id - 获取剧集详情
- PUT /episodes/:id - 更新剧集
- DELETE /episodes/:id - 删除剧集
- GET /episodes/:id/stats - 获取剧集统计

### 7.2 Shot API
- GET /episodes/:episodeId/shots - 获取分镜列表
- POST /episodes/:episodeId/shots - 创建分镜
- GET /shots/:id - 获取分镜详情
- PUT /shots/:id - 更新分镜
- DELETE /shots/:id - 删除分镜
- POST /shots/:id/generate - 生成视频

### 7.3 ShotAlternative API
- GET /shots/:shotId/alternatives - 获取备选视频列表
- POST /shots/:shotId/alternatives - 创建备选视频
- PUT /shots/:shotId/alternatives/:id/recommend - 设为推荐
- DELETE /shots/:shotId/alternatives/:id - 删除备选视频

### 7.4 ShotDraft API
- GET /episodes/:episodeId/drafts - 获取草稿列表
- POST /episodes/:episodeId/drafts - 保存草稿
- PUT /drafts/:id - 更新草稿
- DELETE /drafts/:id - 删除草稿

### 7.5 Mention API
- GET /projects/:projectId/mentions - 获取可提及的资源

- 数据库迁移可能失败

### 9.2 业务约束
- 素材库按项目隔离

### 9.3 时间约束
- 项目需要在11天内完成
- 需要分阶段交付

### 9.4 资源约束
- 开发人员数量有限
- 测试资源有限

## 10. 附录

### 10.1 术语表
- **剧集（Episode）**：一个完整的故事单元，包含多个场景
- **场景（Scene）**：剧本中的一个场景，包含地点、时间等信息
- **分镜（Shot）**：一个镜头，对应一个视频片段
- **备选视频（Alternative）**：同一个分镜生成的多个视频版本
- **@提及（Mention）**：在描述中引用资源的功能
