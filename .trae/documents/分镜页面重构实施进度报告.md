## 技术决策

1. **数据库设计**
   - Scene 通过 episode_id 关联 Episode，不直接关联 Project
   - Shot 同时保留 project_id 和 episode_id 便于查询

2. **API 设计**
   - 统一使用 snake_case
   - 响应格式：`{ success: boolean, data?: T, error?: {...} }`

3. **前端架构**
   - 使用 React Query 管理服务器状态
   - API 模块按功能分组
